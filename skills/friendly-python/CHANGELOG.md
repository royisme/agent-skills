## [1.0.0] - 2026-01-09

### Added
- Initial release of Friendly Python skill
- Core design principles (user-friendly + maintainer-friendly)
- 7 code patterns with good/bad examples:
  - Registry vs If-Else
  - Context Manager vs Manual Cleanup
  - Classmethod Constructors vs Flag-based Init
  - Descriptors vs __getattr__ Catch-all
  - Extension Points vs Custom Wrappers
  - Python Idioms vs Callback Style
  - Argparse OOP Commands
- Review checklist for code quality assessment
- Quick reference tables for patterns to use and avoid
- References to original Frost Ming articles
